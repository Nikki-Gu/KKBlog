### SQLite和MySQL的对比

| 对比     | MySQL                      | SQLite                                     |
| -------- | -------------------------- | ------------------------------------------ |
| 存储方式 | 服务端，存储在服务器硬盘上 | 嵌入式，存储在单一文件中                   |
| 并发支持 | 支持高并发                 | 并发支持较弱（只有文件锁的锁，不支持行锁） |
| 扩展性   | 好                         | 差                                         |
| 事务     | 四种事务隔离级别           | 只支持一个写事务                           |
| 约束     | 自动添加主键，not null约束 | 默认不开启主键非空和外键约束               |
| 用户管理 | 支持                       | 不支持                                     |

sqlite只支持ALTERT TABLE的有限子集，不提供对表中属性的重命名，删除，和更新约束等功能，所以当表建错的时候只能重新建表再导入数据；
